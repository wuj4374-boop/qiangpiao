二、软件简介

本软件是一个基于 Python + FastAPI + Playwright 的 Web 自动化学习项目，演示了以下技术原理：

  • Playwright 浏览器自动化控制
  • FastAPI 后端 API 设计
  • WebSocket 实时通信
  • JWT 身份认证
  • 反检测技术（WebDriver 隐藏、指纹伪装等）
  • 前端与后端的交互架构
工作原理: 软件通过 Playwright 控制一个真实的 Chromium 浏览器窗口，自动模拟人工操作（点击、选择、提交），使用你自己的大麦网账号完成购票流程。

三、环境要求

项目 要求 说明
操作系统 Windows 10 / 11 64 位系统
Python 3.8 及以上 推荐 3.10+
内存 4GB 以上 浏览器占用较多内存
网络 可访问大麦网 国内网络即可
浏览器 项目自带 Chromium 无需额外安装
提示: 如果你的电脑没有安装 Python，请先前往 https://www.python.org/downloads/ 下载安装，安装时务必勾选 "Add Python to PATH"。

四、安装步骤

1 下载并解压软件
将整个 damai_bot 文件夹放到你电脑的任意位置（路径中不要包含中文）。
2 确认 Python 已安装

按 Win + R，输入 cmd 回车，输入以下命令：

python --version

如果显示 Python 3.x.x 说明已安装。如果提示找不到命令，请先安装 Python。

3 双击启动脚本

双击 启动抢票系统.bat，脚本会自动：

  • 检测 Python 环境
  • 检查并安装所需依赖（首次运行需要，使用国内镜像加速）
  • 启动后端服务
  • 自动打开浏览器
首次启动较慢: 第一次运行需要安装依赖包（fastapi、uvicorn、playwright 等），请耐心等待。后续启动会直接跳过安装步骤。

五、使用教程

第一步: 启动软件

1 双击 启动抢票系统.bat

在 damai_bot 文件夹中找到 启动抢票系统.bat，双击运行。

会弹出一个黑色命令行窗口，显示启动信息。启动成功后会自动打开浏览器。

第二步: 同意免责声明

2 阅读并同意免责声明

命令行窗口会显示免责声明全文，请仔细阅读。

阅读完成后，输入 agree 并按回车键。

输入其他任何内容将退出程序。

第三步: 登录系统

3 浏览器自动打开，进入登录页面

浏览器会自动打开系统登录页面，使用默认账号登录：

字段 默认值
用户名 admin
密码 admin123
API 地址 http://localhost:8000

直接点击 "登录系统" 按钮即可（默认值已填好）。

第四步: 添加大麦账号

4 进入大麦网账号管理页面

登录后进入"大麦网账号管理"页面。

点击 "添加账号" 按钮，创建一个新的账号卡槽。

可以添加多个账号，多账号会同时抢票，提升成功率。

第五步: 扫码登录大麦网

5 扫描二维码登录你的大麦账号

在账号卡片上点击 "扫码登录" 按钮。

系统会自动打开大麦网登录页面，页面上会显示二维码。

用手机打开 大麦APP 或 支付宝，扫描二维码完成登录。

登录成功后，账号卡片会显示你的昵称和"已登录"状态。

至少一个账号登录成功后，"进入抢票" 按钮会变为可点击状态。

注意: 扫码登录使用的是你的真实大麦网账号。登录后的 Cookie 仅保存在你的电脑本地，不会上传到任何服务器。

第六步: 配置抢票任务

6 填写抢票配置信息

点击 "进入抢票" 后，进入主控制面板。在左侧填写以下信息：

字段 说明 示例
演出链接 粘贴大麦网演出页面的完整链接，或直接输入演出ID https://m.damai.cn/shows/item.html?id=123456
票数 需要购买的票数（1-6） 2
城市 演出所在城市 上海
观演人 每行一人，格式：姓名,身份证号,手机号 张三,110101199001011234,13800138000
场次 目标场次时间，多个用逗号分隔 19:30
票价 目标票价，多个用逗号分隔（按优先级排列） 980,1280
抢票次数 最大重试次数，默认 999 999
刷新间隔 每次尝试的间隔（毫秒），越小越快 200
开票时间 可选，设置后会等到指定时间自动开始抢 2026-05-10 10:00
选择抢票账号 勾选要参与抢票的已登录账号 默认全选
快速获取演出链接: 在大麦APP或网页中找到目标演出，复制分享链接，粘贴到"演出链接"输入框即可。系统会自动解析演出ID。

第七步: 开始抢票

7 点击"开始抢票"

确认配置无误后，点击 "开始抢票" 按钮。

系统将自动执行以下操作：

  1. 打开浏览器访问演出页面
  2. 自动选择场次和票价
  3. 点击购买按钮
  4. 自动处理验证码（如有）
  5. 自动选择观演人
  6. 提交订单

右侧的实时日志会显示每一步的操作状态。

如果设置了开票时间，系统会先等待，到时间后自动开始。

多账号模式: 如果勾选了多个账号，所有账号会同时抢票。一旦某个账号抢票成功，其他账号的任务会自动取消。

第八步: 完成支付

8 抢票成功后，请尽快完成支付

当系统提示"抢票成功"时，会弹出成功窗口，显示：

  • 演出名称
  • 场次信息
  • 票价
  • 订单链接

请立即在手机上打开大麦APP完成支付！订单有时间限制，超时会自动取消。

你也可以点击"复制链接"，在浏览器中打开订单页面进行支付。

重要: 软件只负责抢到订单，付款需要你自己完成。请务必在规定时间内完成支付，否则订单会被取消。

七、常见问题

Q: 双击 bat 文件没反应怎么办？
A: 右键点击 bat 文件，选择"以管理员身份运行"。如果仍然没反应，检查是否安装了 Python 并添加到了 PATH 环境变量。
Q: 依赖安装失败怎么办？
A: 打开命令行，手动执行以下命令：
cd damai_bot 目录路径
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
Q: 扫码登录后没有反应？
A: 系统每3秒检测一次登录状态，最多等待2分钟。如果超时未检测到，请重新点击"扫码登录"。确保你是用大麦APP或支付宝扫码，而不是微信。
Q: 抢票一直不成功？
A: 抢票成功率受多种因素影响：网络延迟、服务器负载、票务库存、验证码等。建议：
1. 使用有线网络而非WiFi
2. 减小刷新间隔（但不要低于50ms，否则可能被封）
3. 使用多账号同时抢票
4. 设置开票时间，提前准备好
Q: 浏览器打开了但是没有自动操作？
A: 可能是大麦网页面结构更新导致选择器失效。检查实时日志中的错误信息。如果是验证码问题，尝试手动完成验证码后让系统继续。
Q: Cookie 过期了怎么办？
A: 回到账号管理页面，重新点击"扫码登录"。Cookie 有效期通常为几小时到一天不等。建议在抢票前重新登录一次确保 Cookie 有效。
Q: 能同时抢多个不同演出吗？
A: 不支持。每个任务只能配置一个演出。如需抢多个演出，可以打开多个浏览器实例（不推荐，可能影响性能）。
Q: 软件会泄露我的账号密码吗？
A: 不会。所有登录信息（Cookie）仅保存在你电脑本地的 cookies 文件夹中，不会上传到任何服务器。软件本身也不收集任何个人信息。

八、故障排除

8.1 启动类问题

问题原因解决方案
找不到 Python 未安装 Python 或未添加到 PATH 安装 Python 并勾选 "Add to PATH"，或手动编辑 bat 文件指定 Python 路径
端口 8000 被占用 其他程序占用了 8000 端口 关闭占用端口的程序，或修改 start_web.py 中的 PORT 值
依赖安装超时 网络问题 检查网络连接，或使用代理。也可以手动下载 whl 文件安装

8.2 登录类问题

问题原因解决方案
扫码后无反应 使用了微信扫码 必须使用大麦APP或支付宝扫码
登录状态频繁失效 Cookie 过期 抢票前重新扫码登录
浏览器打不开登录页 网络问题或 Chromium 异常 检查网络连接，删除 ms-playwright 文件夹后重新启动

8.3 抢票类问题

问题原因解决方案
页面找不到购买按钮 大麦网页面更新 等待软件更新，或手动在浏览器中操作
验证码无法自动处理 验证码类型不支持 尝试手动完成验证码
提交订单失败 库存不足或网络超时 系统会自动重试，持续观察日志

九、注意事项

使用前必读:
  • 本软件为学习研究专用，请勿用于商业或非法用途
  • 请遵守大麦网的用户协议和服务条款
  • 频繁的自动化操作可能导致账号被风控，请合理使用
  • 抢票成功后务必在规定时间内完成支付
使用建议:
  • 抢票前先用"检测全部"确认所有账号登录状态正常
  • 使用有线网络连接，减少网络延迟
  • 开票前5分钟启动软件并完成登录
  • 观演人信息提前填好，避免抢票时手忙脚乱
  • 刷新间隔不要设置太小（建议不低于 100ms），避免触发风控
  • 抢票过程中不要手动操作浏览器窗口
隐私安全:
  • 所有数据仅存储在本地电脑，不联网上传
  • Cookie 文件位于 backend/cookies/ 目录
  • 如需清除登录信息，删除该目录下的 JSON 文件即可
  • 截图保存在 screenshots/ 目录（如已启用）
